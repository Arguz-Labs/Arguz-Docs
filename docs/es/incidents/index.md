# Errores e incidentes

Arguz vincula fallas operativas con los cambios y workloads alrededor de ellas para que la investigacion tenga contexto de rollout.

## Que puedes revisar

- errores activos
- errores historicos
- revisiones con issues asociados
- alcance por workload
- cobertura de politicas de alerta

## Fuentes de error dentro del alcance documentado

- fallas de pods como `CrashLoopBackOff`
- `OOMKilled`
- inestabilidad por reinicios
- ejecuciones fallidas de CronJobs y sus logs
